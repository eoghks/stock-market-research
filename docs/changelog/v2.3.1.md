# v2.3.1 변경 사항 (2026-04-27)

## 버그 수정 / 기능 추가 (PATCH)

### 이메일 PDF 자동 발송

보고서 생성 완료 후 PDF를 이메일로 자동 발송하는 기능 추가.

#### 동작 조건

- `config/email_config.json` 파일이 존재할 때만 발송 시도 (없으면 조용히 skip)
- LibreOffice가 설치되어 PDF가 생성된 경우에만 첨부 발송
- PDF 생성 실패 시 이메일도 skip (DOCX는 항상 저장됨)

#### 추가 파일

| 파일 | 설명 |
|------|------|
| `scripts/email/send_report.js` | nodemailer 기반 SMTP 발송 모듈 |
| `config/email_config.json` | SMTP 인증 정보 (gitignore, 커밋 안 됨) |
| `config/email_config.example.json` | 설정 예시 파일 |

#### 지원 메일 서버

| 서비스 | SMTP 호스트 | 포트 |
|--------|------------|------|
| 네이버 | smtp.naver.com | 465 |
| Gmail | smtp.gmail.com | 465 |
| 카카오 | smtp.kakao.com | 465 |
| 다음 | smtp.daum.net | 465 |

#### 이메일 구성

- **첨부**: PDF (필수) + DOCX (선택)
- **제목**: `[증시 보고서] YYYYMMDD HHMM`
- **본문**: 날짜 + 자동 발송 안내 문구

#### generate_report.js 변경

PDF 변환 블록 바로 뒤에 이메일 발송 로직 추가 (14줄).
`generatedPdfPath` 변수로 PDF 경로를 발송 모듈에 전달.
