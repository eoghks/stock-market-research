# v2.3.2 변경 사항 (2026-04-28)

## 버그 수정 (PATCH)

### 날짜 고정 문제 수정 — KST 실행 시각 기준으로 동적 계산

#### 문제
`raw.base_date`(크롤링 JSON의 날짜)를 그대로 사용하여 테스트 데이터나 오래된 파일
실행 시 날짜가 고정되는 문제 발생. 이메일 제목도 파일명에서 날짜를 추출하여 불안정.

#### 수정 내용

**`scripts/generate_report.js`**
- `raw.base_date` 무시 — 항상 **실행 시각 기준 KST**로 날짜 계산
- `KR_SESSION_DATE` (한국 장 기준일) / `US_SESSION_DATE` (미국 장 기준일) 별도 산출
- 커버 페이지에 두 날짜 모두 표시:
  ```
  생성: 2026-04-28 19:00 KST
  🇰🇷 한국 장: 2026년 04월 28일(월)   🇺🇸 미국 장: 2026년 04월 25일(금)
  ```

**KST 기준 시장 세션 날짜 판정 로직**

| KST 시각 | 한국 기준일 | 미국 기준일 |
|---|---|---|
| 09:00 ~ 15:30 | 오늘 (장중) | 어제 (전일 ET) |
| 15:30 ~ 익일 06:00 | 오늘 (마감) | 어제 (전일 ET) |
| 06:00 ~ 09:00 | 어제 (전일) | 어제 (오늘새벽 ET) |
| 00:00 ~ 06:00 | 어제 (전일) | 2일 전 (ET) |

**`scripts/email/send_report.js`**
- 이메일 제목 날짜를 파일명 파싱 방식에서 **KST 현재 시각**으로 변경
- 형식: `[증시 보고서] 2026-04-28 19:00 KST`
